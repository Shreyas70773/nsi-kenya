# SEO Audit & Optimization Plan
*Date: 2026-05-26 · Scope: Phase 1 + Tier 2 pages, post-Case-Studies removal*

## Methodology

Audited against the keyword strategy in the foundation spec (Section 5),
the SERP snapshot, and current Google ranking practice for Kenya B2B
industrial queries.

For each page, checked:

1. **Title tag** — 50 to 60 chars total (including ` · North Star Impex Kenya`
   suffix the root layout appends), primary keyword in the first 30 chars,
   no fluff, no duplication across pages.
2. **Meta description** — 140 to 160 chars, primary keyword in the first
   half, value prop or differentiator, soft CTA where natural.
3. **H1** — one per page, contains keyword naturally.
4. **Heading hierarchy** — H1 → H2 → H3, no skipped levels.
5. **OpenGraph image** — per page, using the page hero so the
   Twitter / WhatsApp / LinkedIn link preview matches the page.
6. **JSON-LD coverage** — Organization + BreadcrumbList everywhere
   (handled by primitives); page-type-specific schemas (Product, Service,
   Article, SoftwareApplication, FAQPage) where applicable.
7. **Internal linking density** — every page links to at least 2 related
   product or industry pages.
8. **Image alt text** — non-empty on all `<Image>` tags except decorative
   ones marked `alt=""`.
9. **Word count** — flagship pages ≥ 600 words.
10. **Kenya-specific signals** — "Kenya" or "Nairobi" appears at least
    twice per page, NEMA / EMCA / KEBS where relevant.

## Structural recommendations applied this sweep

1. **Every page metadata block** gets `openGraph.images = [{ url: heroImage }]`
   so link previews on Twitter, WhatsApp, LinkedIn, and Slack carry the
   page hero image. Title and description in `openGraph` mirror the page
   title and description.

2. **`metadata.keywords` array** added per page with the primary keyword
   plus 4 to 6 close variants. Modern search engines ignore the
   `<meta name="keywords">` tag, but a few smaller engines and AI
   crawlers still consume it; it costs nothing.

3. **FAQPage JSON-LD added** to five flagship pages:
   - `/products/tanks/stainless-steel/` (304 vs 316L, lead time, finish)
   - `/products/tanks/zinc-alum/` (service life, vs carbon steel, install time)
   - `/industries/etp-water-treatment/` (NEMA limits, retrofit cost, lead time)
   - `/products/iot/` (NB-IoT vs LoRa, data hosting, battery life)
   - `/products/silos/grain-storage/` (aeration, level instrumentation, capacity sizing)

   Each FAQ block uses spec-section-5 People-Also-Ask style queries that
   Kenyan operators actually search.

4. **Internal-link density check** on flagship pages: every product page
   now links to at least its industry pages, the IoT page, and one
   adjacent product. Cross-linking from industries to products already
   in place.

5. **Removed remaining `/case-studies/` orphan refs** from sitemap (was
   already done), llms.txt (was already done), homepage (was already
   done), about page (was already done), success page (was already done).
   Header NAV had been missed in the previous sweep; fixed in this one.

## Things this audit did NOT do (next-phase work)

- **Build /compare/zinc-alum-vs-carbon-steel/** (Five-Wins #3, Tier 3
  programmatic SEO). High SEO value, separate task.
- **People-Also-Ask harvesting per Tier-1 keyword.** The 5 FAQ pages we
  add this sweep are based on common operator questions, not on a
  systematic SERP scrape. Worth running a fresh Firecrawl pass for that.
- **Backlink outreach.** KAM, KEPSA, KEPSA, NEMA-adjacent directories
  are the targets per spec section 5; the outreach itself is not a code
  change.
- **Schema markup for `OfferCatalog` on /products/`.** Optional;
  Product JSON-LD on individual product pages is sufficient for the
  Tier-1 surfaces.
